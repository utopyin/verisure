Goal:
Build `packages/domain` as a pure package with no Cloudflare, Better Auth, storage, HTTP, or RPC implementation dependencies.

Scope:
- Define domain DTOs for installations, arm state, alarm mutation result, device statuses, connection status, API-token summaries, and shortcut export results.
- Define alarm mode values used by dashboard and Shortcut REST: at minimum `DISARMED` and `ARMED_AWAY`; keep `ARMED_HOME` available for dashboard support.
- Define structured Verisure/domain errors used by server and RPC mapping.
- Implement pure Verisure GraphQL operation builders inspired by the Python implementation: fetch installations, arm state, arm away/home, disarm, door/window, climate, smart locks, smart plugs, event log if included in initial surface.
- Include cookie parsing/serialization helpers if they are pure and useful for server tests.

Testing/validation:
- Colocated tests assert exact GraphQL operation payloads against the Python source expectations.
- Tests cover alarm mode mapping and error classification helpers.
- The package must not import Cloudflare, Better Auth, TanStack, D1, or server implementation modules.

Notes:

- If uncertain about a GraphQL field, copy the Python payload first and leave a small note for later refinement.
