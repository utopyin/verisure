Goal:
Build the server-side config, email, and Better Auth services in `packages/server`.

Scope:
- Implement `RuntimeConfigLive` using Effect Config for `BETTER_AUTH_SECRET`, `CREDENTIAL_ENCRYPTION_KEY`, and optional `TOKEN_PEPPER`.
- Define cloud-agnostic `EmailService` with send and magic-link methods.
- Implement `CloudflareEmailLive` as an Alchemy infrastructure layer that declares/binds Cloudflare SendEmail and implements `EmailService`.
- Implement `BetterAuthService` that exposes Better Auth handler/fetch plus session lookup helpers.
- Build Better Auth config in an Effect that yields `EmailService` and D1/Drizzle dependencies; email sending must not leak into identity/auth service contracts.
- Mounting happens later in the API worker task; this task builds reusable services.

Testing/validation:
- Colocated tests use a stub EmailService to verify magic-link sending is invoked by Better Auth config behavior where feasible.
- Config tests verify missing required config fails clearly and redacted values are not logged.
- Keep Cloudflare binding details inside the live layer.

Notes:
- Do not create a generic `IdentityService` that hides email responsibilities. Better Auth handles auth; EmailService handles email.
