Goal:
Build per-credential Verisure session storage using a Durable Object, replacing Python's cookie file.

Scope:

- Implement `VerisureSessionObject` export in `apps/api` or a server package as appropriate.
- Implement `VerisureSessionStore` service that uses one object per `verisureCredentialId` / CurrentCredential.
- Store session snapshots: cookies, trust token, preferred base URL, authenticated/expiry times, and temporary MFA cookies.
- Provide `withCredentialLock` semantics so login/refresh/cookie mutation is serialized for a credential.
- Bind the Durable Object namespace through Alchemy and the API Worker.
