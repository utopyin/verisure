Goal:
Build `packages/rpc-contract` as the single typed contract imported by both `apps/web` and `apps/api`.

Scope completed:
- Added shared Effect Schema DTOs for dashboard user/session, credentials, installations, alarm state/results, device status reads, shortcut exports, and API token summaries.
- Added typed schema errors for dashboard RPC failures: unauthorized, forbidden, credential not found, installation not found, invalid input, and Verisure upstream errors using the domain error vocabulary.
- Added `CurrentUser` context tag and `DashboardAuthMiddleware` declaration only; Better Auth implementation remains out of this contract package.
- Added Dashboard RPC groups:
  - `AuthRpcs`
  - `CredentialRpcs`
  - `InstallationRpcs`
  - `AlarmRpcs`
  - `DeviceRpcs`
  - `ShortcutRpcs`
- Added combined authenticated `DashboardRpcs` contract with dashboard auth middleware applied to every RPC.
- Exported contract schemas, errors, middleware, RPC groups, and `DashboardRpcs` type helpers from `packages/rpc-contract/src/index.ts`.
- Added minimal imports in `apps/api/src/Rpc/DashboardHandlers.ts` and `apps/web/src/rpc/client.ts` to confirm API and web can import the shared contract without circular dependencies.
- Added package `test` script for colocated contract tests.

Testing/validation:
- Colocated tests verify representative schema decoding, alarm mode validation, typed upstream error decoding, expected RPC group tags, combined contract size, and auth middleware presence on each RPC.
- `bun run --filter @verisure/rpc-contract check:types` passes.
- `bun run --filter @verisure/rpc-contract test` passes.
- Full repo `bun run check:types` passes.
- Full repo `bun run test:turbo` passes.
- Full repo `bun run test` passes.

Notes:
- This is contract-only. No RPC handlers, Better Auth logic, TanStack code, D1, Cloudflare, or server implementation modules are imported into `packages/rpc-contract`.
- RPC tags are prefixed by group, e.g. `Alarm.SetAlarmMode` and `Shortcut.ExportShortcut`, to keep handler/client names stable and explicit.
