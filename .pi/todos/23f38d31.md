Goal:
Build `packages/rpc-contract` as the single typed contract imported by both `apps/web` and `apps/api`.

Scope:
- Define Effect Schema DTOs and typed schema errors for dashboard calls.
- Define Dashboard RPC groups for auth/session, credentials, installations, alarm commands, device reads, and shortcut export/token management.
- Define middleware declarations for dashboard auth only; do not implement Better Auth here.
- Export contract types and helpers needed by client and server.

Dependencies:
- Depends on `TODO-1b7a729e` for workspace scaffolding.
- Depends on `TODO-69ba5e95` for domain DTOs and error vocabulary.

Testing/validation:
- Colocated contract tests verify schemas encode/decode representative payloads.
- The package must not import `packages/server`, `packages/db`, Cloudflare, Better Auth, or TanStack.
- Confirm server and web can import the contract without circular dependencies.

Notes:
- This is contract-only. Handler implementation is a separate API/server task.
- Prefer explicit typed RPC errors over generic JSON error blobs.
