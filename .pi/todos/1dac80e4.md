- `docs/architecture.md` sections: Service model, Data model, Key design decisions

Goal:
Build the server utilities and context tags that keep credential-scoped service APIs clean and secure.

Scope:

- Implement `CredentialCrypto` using Web Crypto-compatible AES-GCM or another workerd-safe approach, keyed by `CREDENTIAL_ENCRYPTION_KEY` from RuntimeConfig.
- Encrypt/decrypt Verisure email, password, and optional PIN.
- Define `CurrentUser`, `CurrentCredential`, and `CurrentInstallation` Effect context tags.
- Implement credential authorization/scope helpers that verify ownership and provide context tags.
- Ensure decrypted credential material stays in the smallest possible scope and is never logged.

Dependencies:

- Depends on `TODO-a3b66ad4` for RuntimeConfig.
- Depends on `TODO-11420995` for credential repository shapes.
- Depends on `TODO-69ba5e95` for domain credential/status types.

Testing/validation:

- Colocated tests verify encryption/decryption round trips and wrong-key failures.
- Tests verify ownership checks reject credentials for another user.
- Tests verify context providers make services callable without passing repeated ids.

Notes:

- Runtime is workerd; avoid Node-only crypto APIs.
- This task unlocks Verisure services, RPC scope middleware, and Shortcut REST auth.
